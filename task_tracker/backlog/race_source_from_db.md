# Источник гонки (AM/Athletex) — отдавать из БД, убрать хардкод на фронте

> Статус: backlog
> Дата: 2026-06-10
> Тип: tech debt / рефакторинг

## Проблема

Фильтр каталога «Athletex / Алматы Марафон» работает по **захардкоженному списку**
6 AM-id во фронте (`AM_RACE_IDS` в `frontend/src/types/races.ts::getRaceSource`).

Это второй источник правды: `runners.source` (`am`/`athletex`) УЖЕ есть в БД на
каждом бегуне, но API `list_races` его не отдаёт, и фронт угадывает источник по
списку id. Добавишь новую AM-гонку — надо не забыть дописать её id во фронт, иначе
она уедет в Athletex.

## Чистое решение

1. Backend: `list_races` (`api/v1/routes/races.py`) — добавить `source` в `RaceSchema`,
   вычислять из `runners.source` результатов гонки (один запрос: race_id → source).
2. Frontend: убрать `AM_RACE_IDS` / переписать `getRaceSource` на чтение `race.source`
   из ответа API.

Тогда: один источник правды (БД), новые AM-гонки определяются автоматически,
`tau_jarys` и подобные не требуют ручной пометки.

## Заметка
Бейдж trail/road на карточке (`RACE_TYPE_BY_ID`) — отдельная ось, к источнику
отношения не имеет. Его решать отдельно (или убрать).
