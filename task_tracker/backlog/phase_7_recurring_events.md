# Фаза 7: Постоянные активности (регулярные события)

## Цель

Добавить в систему регулярные беговые активности Алматы (кофераны, дайхарды и т.п.) — события, которые проходят еженедельно/ежемесячно. Привязать их к клубам. Стать источником правды для регулярных событий.

## Контекст

- Кофераны (coffeerun) — еженедельные утренние пробежки, несколько групп в Алматы
- Дайхарды (diehard) — регулярные интенсивные тренировки
- У этих событий нет формальных результатов/тайминга — люди просто приходят и бегут, трекают в Strava самостоятельно
- Эти события по сути принадлежат клубам/группам

## Сценарий создания

1. **Выяснить расписание**: когда, где, какой формат (дистанция, набор)
2. **Создать клуб** для организаторов (если ещё нет из фазы 5-6)
3. **Создать recurring events** привязанные к клубу
4. **Показать в календаре**: "Каждую субботу — Coffeerun, 7:00, Медеу"
5. **Передать клуб** реальным организаторам (фаза 6) для дальнейшего управления

## Данные

### Формат recurring event
- Название
- Клуб-организатор
- Расписание (day of week, время, частота)
- Локация (название + координаты)
- Тип (coffeerun / training / diehard / open run)
- Примерная дистанция и набор (если фиксированные)
- Контакт (Telegram-группа, ссылка)

### Хранение
Варианты:
- Расширить `content/races/races.yaml` секцией `recurring_events`
- Или отдельный файл `content/events/recurring.yaml`
- Или хранение в БД через ayda_run (если нужна динамика)

## Matching активностей (будущее)

Из обсуждения: можно матчить Strava-активности участников клуба по:
- Времени проведения события (день недели + время ±1 час)
- Примерной дистанции и набору
- Геолокации старта

Это даёт возможность автоматически считать "кто пришёл" и строить статистику/лидерборды внутри клуба — аналог того, как это делается в ayda_run.

**Важно:** Matching по Strava — только внутри клуба (Community App). Для внешних — только Garmin/manual.

## Интеграция

### Бот
- Команда для просмотра ближайших регулярных событий
- "Что бегут на этой неделе в Алматы?"

### Веб-дашборд (фаза 4)
- Секция "Регулярные события" в календаре
- На странице клуба — расписание его активностей

### ayda_run
- Админ клуба управляет расписанием через TMA
- Запись на тренировку (существующая функциональность ayda_run)
