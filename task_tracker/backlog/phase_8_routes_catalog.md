# Фаза 8: Каталог маршрутов и вершин

## Цель

Создать базу маршрутов и вершин Алматы: GPX, грейды сложности, расчёт km-effort, типичное время, оценка готовности. Это ядро публичного слоя (Слой 1 из agent_brief.md) — контент, который привлекает трафик и даёт ценность без регистрации.

## Контекст

Из agent_brief.md (секции 2, 3.5, 4, 6):

**Целевой каталог:** 15-20 маршрутов/вершин Алматы.

**Примеры:**
- Вершины: Пик Фурманова, Букреева, Кумбель, БАП (Большой Алматинский Пик)
- Маршруты: Кок-Жайляу, Бутаковка, Кимасар

**Источники GPX:**
- Собственные пробежки Рената
- GPX от тренеров SRG, RUNFINITY
- Специально записанные треки
- **НЕ из Strava** — не извлекаем маршруты из активностей юзеров

## Что содержит каталог (для каждого маршрута)

### Базовые данные
- Название, описание, локация
- GPX файл (эталонный трек)
- Расстояние, набор/сброс высоты
- Старт/финиш координаты и высоты

### Расчётные метрики (используем существующий gpx_predictor)
- **km-effort** — эквивалентная плоская дистанция (формула ITRA: дистанция + набор × коэффициент)
- **Грейд сложности** — адаптация ITRA Mountain Level (0-12) → цветовая шкала:
  - 🟢 Зелёный (ML 0-3): доступно начинающим
  - 🟡 Жёлтый (ML 4-6): средний уровень, нужен опыт
  - 🟠 Оранжевый (ML 7-9): продвинутый, длинный день
  - 🔴 Красный (ML 10-12): экспертный, альпинистский опыт
- **Типичное время** для разных уровней (хайкер / бегун / элита) — базовый gpx_predictor

### ITRA-производные (секция 4 agent_brief.md)
- **Mountain Level** (0-12): соотношение набор/дистанция + средняя высота
- **Finisher Level**: минимальный уровень подготовки для безопасного прохождения
- **Estimated Race Score**: при наличии времени юзера — оценка уровня

### Оценка готовности ("Готов ли ты?") — секция 6 agent_brief.md

**Без авторизации:**
- Чеклист для каждого грейда: "Если ты можешь пройти 15 км с набором 1000м за X часов — ты готов"
- Рекомендации по безопасности, окно выхода

**С авторизацией (после подключения Strava/Garmin):**
- На основе backfill данных:
  - Максимальный набор высоты за одну активность (3 месяца)
  - Максимальная дистанция
  - Частота горных выходов
  - Средний темп на подъёмах
- Результат: 🟢 готов / 🟡 с оговорками / 🔴 не готов + "сначала сходи на маршрут X"

## Интеграция

### Калькулятор времени (без регистрации)
- "Введи темп на плоскости → получи прогноз на маршрут"
- Существующий gpx_predictor в базовом режиме

### Персональный прогноз (с авторизацией)
- Strava/Garmin backfill → персональный gradient-pace профиль → точный прогноз
- "Твоё предполагаемое время на Пик Фурманова: 4:20"

### Бот
- Команда для просмотра маршрутов
- Прогноз времени на маршрут (как для гонок)

### Веб-портал (фаза 9)
- Страница маршрута: карта, профиль высот, грейд, калькулятор
- SEO: каждый маршрут = отдельная страница → трафик

## Связь с гонками

Некоторые маршруты = дистанции гонок (Alpine Race Skyrunning = подъём на Шымбулак). Нужна возможность связать route с race distance.

## Зависимости

- Независим от фаз 3-7 (можно делать параллельно)
- GPX файлы нужно получить от Рената
- Для персонализации — нужна фаза 2.5 (Strava через ayda_run API)

## Что существует в коде

- `backend/app/features/gpx/` — GPX парсер, сегментация
- `backend/app/features/hiking/calculators/` — Tobler, Naismith калькуляторы
- `backend/app/features/trail_run/calculators/` — GAP calculator
- `backend/app/services/prediction.py` — оркестратор прогнозов
- `backend/app/shared/` — geo, elevation, formulas

Каталог маршрутов может переиспользовать всю эту инфраструктуру.
