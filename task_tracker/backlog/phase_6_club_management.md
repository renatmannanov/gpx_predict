# Фаза 6: Регистрация и управление клубами

## Цель

Дать возможность админам клубов регистрироваться на платформе и управлять своим клубом: добавлять/удалять участников, корректировать данные. Регистрация через ayda_run.

## Контекст

- Клубы уже извлечены из результатов гонок (фаза 5) — есть "пассивные" клубы
- Нужно превратить их в "активные" — с владельцем и админкой
- ayda_run — существующий Telegram Mini App с 42 юзерами, SRG + RUNFINITY уже там
- ayda_run имеет статус Strava Community Application — можно показывать данные участников друг другу внутри клуба

## Сценарий

1. **Мы создаём клуб** на основе данных из результатов гонок (фаза 5)
2. **Находим реального админа** (тренера/организатора клуба)
3. **Передаём управление**: админ получает возможность:
   - Подтвердить/отклонить участников (из данных гонок)
   - Добавить новых участников (не из гонок)
   - Удалить неактуальных
   - Редактировать описание клуба
4. **Клуб становится "верифицированным"** — данные проверены админом

Пример с кофераном/дайхардом: мы сами создаём клуб "Coffeerun Almaty", наполняем данными, потом передаём организаторам.

## Архитектура

Регистрация и управление клубами — **фича ayda_run**, не gpx_predictor:
- ayda_run владеет OAuth и пользовательскими данными
- ayda_run = Community Application → может показывать данные участников тренеру
- gpx_predictor получает данные о клубах через API ayda_run (или из общей модели данных)

### Роли
- **Владелец клуба** — создатель, полный доступ
- **Админ** — управление участниками (тренер)
- **Участник** — видит дашборд клуба, лидерборды

### Что нужно от ayda_run
- API для CRUD клубов
- API для управления участниками
- Механизм "передачи" клуба (мы создали → передали реальному админу)

## Интеграция с gpx_predictor

gpx_predictor потребляет данные о клубах для:
- Показа клубной статистики по гонкам
- Лидербордов внутри клуба (данные Strava — через Community App)
- Прогнозов для участников клуба

Способ интеграции: API ayda_run → gpx_predictor (как решено в архитектуре — Вариант D).

## Зависимости

- Фаза 5 (клубы извлечены из результатов)
- ayda_run API (нужны endpoints для управления клубами)

## Что НЕ делаем

- Веб-интерфейс для управления (это будет в фазе 9 — портал)
- Пока управление через Telegram Mini App (ayda_run) или через бота
