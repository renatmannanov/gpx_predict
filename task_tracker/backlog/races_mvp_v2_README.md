# Ayda — Races MVP v2

## Контекст

**Полный vision:** см. `agent_brief.md` (оригинальный документ, read-only reference)

## Принятые архитектурные решения

Полный список — в `agent_brief.md`, секция 0. Ключевые:

1. **Контент отдельно от кода:** `content/races/`, `content/routes/` — данные; `backend/app/features/` — код
2. **Сначала бот, потом веб:** MVP через Telegram-бота (уже работает), веб-дашборд — следующий шаг
3. **Strava OAuth через ayda_run (Вариант D):** ayda_run — единственный хозяин OAuth, gpx_predictor получает токены через API
4. **Результаты гонок — файлы, не БД:** JSON в `content/races/results/`, ~20 файлов, БД не нужна
5. **Splits — нормализованная таблица:** `StravaActivitySplit` с отдельными полями, не JSONB
6. **Портал внутри gpx_predict:** Веб-портал разворачивается в gpx_predict. С ayda_run — обмен через API, не shared DB
7. **Лидерборды клубов — через ayda_run Community App:** Strava разрешает показывать данные внутри клуба

## Strava API ограничения (важно для фаз 5-7, 9)

- Данные юзера показывать ТОЛЬКО этому юзеру (кроме Community Applications)
- Community Application (ayda_run): можно показывать данные участников друг другу внутри клуба
- Агрегации из Garmin/manual данных — без ограничений
- Rate limits: 600 req/15min, Athlete Capacity: 999
- Подробности: `agent_brief.md`, секции 3.1 и 10
