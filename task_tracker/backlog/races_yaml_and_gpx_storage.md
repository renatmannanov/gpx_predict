# Races: YAML структура и GPX хранение — план улучшений

**Создано:** 2026-02-24
**Контекст:** Обсуждение при планировании Фазы 2 races MVP

---

## 1. Разделение races.yaml на отдельные файлы

**Проблема:** Единый `content/races/races.yaml` будет расти.
- 21 гонка/год × ~15 строк = ~315 строк
- С историей editions за 3-4 года — 500+ строк
- Неудобно редактировать, легко ошибиться

**Решение:** Один YAML на гонку:
```
content/races/
  catalog/
    alpine_race.yaml
    tengri_ultra.yaml
    irbis_race.yaml
    ...
```

`RaceCatalog` читает все `*.yaml` из папки `catalog/`.

Каждый файл ~30-50 строк, легко редактировать.

**Триггер:** Когда в каталоге > 10 гонок.

---

## 2. GPX хранение — структура папок

**Проблема:** Плоская папка `content/races/gpx/` не масштабируется.
- 7 гонок × 2-3 дистанции × возможные изменения трассы по годам
- Через 2-3 года: 50-60 файлов в одной папке
- Непонятно какой файл к какой гонке относится

**Решение:** Папки по гонкам:
```
content/races/gpx/
  alpine_race/
    skyrunning.gpx              # текущая/дефолтная трасса
    skyrunning_2024.gpx         # если в 2024 была другая трасса
  tengri_ultra/
    ultra_70.gpx
    marathon_42.gpx
  irbis_race/
    trail_20.gpx
```

В YAML:
```yaml
distances:
  - id: skyrunning
    gpx_file: skyrunning.gpx     # дефолтная трасса

editions:
  - year: 2024
    gpx_overrides:                # если трасса отличалась
      skyrunning: skyrunning_2024.gpx
```

`RaceCatalog.get_gpx_path()` ищет в `content/races/gpx/{race_id}/`.

**Альтернатива:** Хранить GPX в БД с тегом `source=race` (единое хранилище с пользовательскими GPX). Плюс: единый API для работы с GPX. Минус: больше кода, GPX гонок — статический контент.

**Триггер:** Когда появятся первые изменения трассы между годами, или > 20 GPX файлов.

---

## 3. Автоматизация YAML из CLAX парсера

**Проблема:** При добавлении новой гонки нужно руками:
1. Парсить CLAX → JSON (автоматически)
2. Класть GPX в папку (руками)
3. Заполнять races.yaml (руками)

**Решение:** `parse_race.py` при первом парсинге гонки:
- Создаёт/обновляет запись в YAML автоматически
- Имена дистанций берёт из CLAX
- GPX файл — единственное что нужно положить руками и указать имя

**Триггер:** Когда добавим > 5 гонок и ручное заполнение начнёт утомлять.
