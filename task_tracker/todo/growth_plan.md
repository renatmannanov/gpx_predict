## Git-стратегия

```
main
 └── feature/step-1-am-data          ← общая ветка шага 1
      ├── step-1a-audit-aliases       ← фаза A: аудит + таблицы алиасов
      ├── step-1b-transliteration     ← фаза B: транслитерация
      ├── step-1c-pilot-am            ← фаза C: пилот одной гонки AM
      └── step-1d-mass-load           ← фаза D: массовая загрузка + деплой
```

**Процесс:**
1. Создать ветку этапа (например `step-1a-audit-aliases`) от `feature/step-1-am-data`
2. Сделать работу → показать → проверяем вместе
3. Коммит → мёрдж в `feature/step-1-am-data`
4. Следующий этап — новая ветка от `feature/step-1-am-data`
5. По завершению всего шага — мёрдж `feature/step-1-am-data` → `main`
