## Фокус

**Сейчас:** ayda.run = портал с результатами гонок для бегунов Алматы.
**Потом:** экосистема для бегунов (портал + Mini App + аналитика). Но пока контент только про портал.

Mini App, трекер тренировок, Strava — **не упоминаем** пока нет ощутимого прогресса.

---

## Принцип

**Каждый значимый коммит = пост.**

Не придумывать контент из воздуха. Делаешь работу → рассказываешь о ней.
Формула: **что сделал + почему это важно для бегуна + CTA (зайди, проверь, найди себя)**.

## Сериал "ayda.build"

Основная контент-линия — **серия постов о том, как мы строим беговой портал для бегунов Алматы.**

Каждый пост — маленький эпизод. Хронология, прозрачность, честность.

### Предыстория (уже опубликовано, ДО системного формата)

| # | Дата | Канал | Что |
|---|------|-------|-----|
| 0.1 | — | TG + Threads + Insta | Знакомство: "Привет, мы ayda" |
| 0.2 | — | Threads + Insta | Карусель: Amangeldy Race — 5 лет, 2003 участника, топ клубов |
| 0.3 | — | Threads + Insta | Карусель: Alpine Race 2026 — 436 финишёров, 77 клубов |
| 0.4 | — | TG | Отчёт: 2 недели беты с клубами (72 человека, 28 регистраций) |

Эти посты — предыстория. Нумерацию `ayda.build #1` начинаем с нового системного поста.

### Зачем сериал
- Building in public привлекает и бегунов, и tech-людей
- Создаёт ощущение "я слежу за проектом" (FOMO)
- Людям интересно быть частью чего-то растущего
- Автору весело — рассказывать о реальной работе, а не выдумывать контент

### Сезоны
- **Сезон 1: "Данные"** — как мы собираем результаты всех гонок Алматы (текущий)
- **Сезон 2: "Видимость"** — SEO, шеринг, страницы клубов
- **Сезон 3: "Сообщество"** — вовлечение клубов, обратная связь, коллабы

### Нумерация
Посты серии маркируются: `ayda.build #1`, `ayda.build #2` и т.д.
Не каждый пост = эпизод сериала. Тип D (вовлечение) и отдельные инсайты — без номера.

## Формат постов

### Тип A: "Сделали" (после коммита/деплоя)
```
[Скриншот с сайта]

Короткий текст: что изменилось + почему это важно.

→ Ссылка на ayda.run
```

### Тип B: "За кулисами" (процесс)
```
[Скриншот с сайта или терминала]

Короткий текст: над чем работаем, зачем.
Без технических деталей — через призму пользователя.
```

### Тип C: "Инсайт из данных" (аналитика)
```
[Скриншот с сайта — страница гонки, профиль, список]

Интересный факт из данных.

→ Все результаты на ayda.run
```

### Тип D: "Вовлечение" (вопрос/опрос)
```
Короткий вопрос к аудитории.
Без картинки — только текст.
Цель — вызвать ответ/реакцию.

Примеры:
- "На какой дистанции бежишь АПМ?"
- "Сколько гонок ты пробежал в 2025?"
- "Какая была твоя первая гонка?"
- "Трейл или шоссе — что тебе ближе?"
```

## Визуал: что реально делать быстро

| Формат | Время | Когда использовать |
|--------|-------|-------------------|
| Скриншот с ayda.run | 2 мин | Типы A, B, C — основной формат |
| Скриншот терминала/данных | 2 мин | Тип B |

**Правило:** обычный пост = скриншот с сайта + текст. Карусели — только для больших событий (макс 1-2 в месяц).

## Правила

1. **Не писать когда нечего показать.** Молчание лучше мусорного поста
2. **Каждый пост — одна мысль.** Не "мы сделали 5 вещей", а "теперь можно X"
3. **Визуал = скриншот с сайта.** Не тратить часы на дизайн
4. **CTA в каждом посте** (кроме типа D). Ссылка на ayda.run или конкретную страницу
5. **Не хвастаться техникой.** "Мы написали транслитерацию" — не интересно. "Теперь можно искать себя на русском" — интересно
6. **Тегать клубы** когда уместно — увеличивает охват
7. **Нумерация сериала** в постах типа A и B: `ayda.build #N`
8. **Три сети одновременно.** Один контент → адаптация под формат канала
9. **Про Mini App не пишем** — только портал с результатами

Не каждый коммит = пост. Только те, которые меняют что-то для пользователя.
