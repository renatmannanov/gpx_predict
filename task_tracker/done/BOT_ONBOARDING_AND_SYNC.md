# Bot Onboarding & Strava Auto-Sync

**Дата:** 2025-01-24
**Статус:** TO DO
**Приоритет:** Высокий

---

## Цель

Создать полноценный onboarding для новых пользователей бота, который:
1. Знакомит с функциональностью
2. Объясняет разницу между стандартным режимом и персонализацией
3. Собирает предпочтения пользователя
4. Проводит через подключение Strava
5. Автоматически синхронизирует данные и уведомляет о прогрессе

---

## Уведомления о прогрессе синхронизации

### Расчёт времени синхронизации (на примере реальных данных)

**Пример пользователя (твои данные):**
- Всего активностей: **192**
- Run/TrailRun: **110**
- Hike/Walk: **14**
- Средняя длина активности: **~10 км** (9.6 сплитов)

**Лимиты Strava API:**
- 200 запросов / 15 минут
- 2000 запросов / день

**Расход на синхронизацию:**

| Операция | Запросов | Комментарий |
|----------|----------|-------------|
| Список активностей | 7 | 192 / 30 per page = 7 страниц |
| Детали + splits | 192 | По одному на активность |
| **Итого на 1 пользователя** | **~200** | Ровно в лимит 15 минут! |

**Сценарий с 100 пользователями (каждый по ~200 активностей):**
- Всего запросов: 100 × 200 = 20,000
- При лимите 2000/день = **10 дней** на полную синхронизацию всех
- При лимите 200/15мин = **~1 пользователь полностью за 15 минут**

**Реалистичный сценарий (батчами по 30 активностей):**

| Batch | Активностей | Запросов | Пользователей за 15 мин |
|-------|-------------|----------|------------------------|
| 1 | 30 | 31 | ~6 |
| 2 | 30 | 31 | ~6 |
| ... | ... | ... | ... |

**Для пользователя с 192 активностями:**

| Этап | Время от подключения | Что происходит |
|------|----------------------|----------------|
| Batch 1 | 15-30 мин | Загружено 30 активностей, профиль начинает заполняться |
| Batch 2-3 | 1-2 часа | 60-90 активностей, уведомление о прогрессе |
| Batch 4-7 | 2-4 часа | Все активности загружены |
| Финал | 4-6 часов | "Синхронизация завершена!" |

### Стратегия уведомлений

| Событие | Уведомление |
|---------|-------------|
| Первый batch загружен (≥10 активностей) | "Загружено X активностей, профиль начинает заполняться..." |
| Профиль впервые заполнен (≥3 категории) | Показать профиль с деталями |
| Есть категории без данных | Показать рекомендации что записать |
| Синхронизация завершена | "Все активности загружены!" |

**Частота уведомлений:** После каждого batch (~30 активностей), но не чаще 1 раза в час.

## Вопросы при загрузке GPX

### Текущий flow (для стандартного режима):

1. Тип маршрута (oneway/roundtrip) — если не loop
2. Уровень опыта
3. Вес рюкзака
4. Размер группы
5. Дети в группе
6. Пожилые в группе

### Новый flow с учётом персонализации:

**Шаг 0: Выбор типа активности (NEW)**
```
Какой прогноз нужен?
[🥾 Хайкинг]  [🏃 Трейлраннинг]
```

**Для Хайкинга:**

| Вопрос | Стандартный режим | С персонализацией |
|--------|-------------------|-------------------|
| Тип маршрута | ✅ Да | ✅ Да |
| Уровень опыта | ✅ Да | ❌ Пропускаем (знаем темп) |

## Автоматическая синхронизация Strava

### Изменения в BackgroundSyncRunner

1. **Приоритет по типу активности:**
   - Смотрим `user.preferred_activity_type`
   - Если "hiking" → сначала Hike/Walk, потом Run/TrailRun
   - Если "running" → сначала Run/TrailRun, потом Hike/Walk

2. **Автоматическая синхронизация splits:**
   - После загрузки активности → сразу грузим её splits
   - Не отдельным шагом, а в том же batch

3. **Автоматический расчёт профиля:**
   - После каждого batch проверяем: достаточно ли данных?
   - Если ≥5 активностей с splits → пересчитываем профиль
   - Создаём событие для уведомления

4. **Создание событий для уведомлений:**
   - `profile_updated` — профиль пересчитан
   - `profile_complete` — все категории заполнены
   - `profile_incomplete` — есть пустые категории
   - `sync_complete` — синхронизация завершена

### Частота синхронизации

- **Новые пользователи:** каждые 5 минут (приоритет)
- **Обычные пользователи:** раз в 6 часов
- **Уведомления:** не чаще 1 раза в день

## Примечания

1. **Уведомления через polling:** Бот проверяет уведомления при любом взаимодействии пользователя + периодически (каждые 5 минут) для активных пользователей.

2. **Rate limits Strava:** 200 запросов / 15 минут. Один пользователь с 50 активностями = ~55 запросов (список + splits). Можно обрабатывать ~3 пользователей за 15 минут.

3. **Два профиля:** Заполняем оба в фоне (если есть данные обоих типов), но уведомляем только о приоритетном. Второй профиль доступен через `/profile`.
