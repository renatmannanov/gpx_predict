# Фаза 5: Клубы из результатов гонок + дашборд клуба

## Цель

Извлечь клубы из спарсенных результатов гонок, построить статистику по каждому клубу, показать лидерборд клубов. Публичные данные — ограничений Strava нет.

## Контекст

- Поле `club` уже есть в `RaceResult` (`backend/app/features/races/models.py`)
- Результаты всех гонок спарсены (фаза 3) — клубы извлекаются из JSON
- Данные о клубах **не полные**: не все участники указывают клуб при регистрации. Это ок для мотивации, но нужен дисклеймер
- В будущем (фаза 6) админы клубов смогут корректировать данные

## Источник данных

Файлы `content/races/results/*.json` → поле `club` у каждого участника.

Примеры клубов из результатов: SRG, RUNFINITY, Nike Running Club, и т.д.

**Проблемы с данными:**
- Разное написание одного клуба ("SRG", "srg", "S.R.G.") → нормализация
- Участник без клуба → не учитывается в клубной статистике
- Участник указал клуб ошибочно → нужна будет возможность коррекции (фаза 6)
- **Был косяк**: при парсинге лишние люди попали в клуб — нужна валидация

## Что строим

### Извлечение клубов
- Пройтись по всем результатам, собрать уникальные клубы
- Нормализация названий (регистр, пробелы, сокращения)
- Список участников каждого клуба (по данным гонок)

### Статистика по клубу (на гонку)
- Количество участников от клуба
- Лучший / медианный / худший результат
- Процент финишёров (если есть DNF)
- Распределение по бакетам ("у SRG 3 человека в топ-25%")

### Статистика по клубу (за сезон)
- Суммарное количество участий по всем гонкам
- Количество уникальных участников
- Средний перцентиль клуба

### Лидерборд клубов
- По конкретной гонке: какой клуб сильнее
- За сезон: суммарный рейтинг
- Сравнение двух клубов (A vs B)

## Интеграция

### Бот
- Кнопка "Клубы" в карточке гонки → лидерборд клубов
- Команда `/clubs` → список клубов + общая статистика
- Поиск: "Покажи статистику SRG"

### Веб-дашборд (фаза 4)
- Секция "Клубы" на странице гонки
- Страница клуба: участники, история выступлений

### API
- Новые endpoints для клубной статистики
- Данные из тех же JSON — без БД

## Зависимости

- Фаза 3 (все гонки спарсены — данные о клубах есть)
- Может идти параллельно с фазой 4 (дашборд)

## Риски

- Качество данных о клубах (нормализация, ошибочные записи)
- Малое количество участников с клубами → статистика может быть нерепрезентативной
- Клубы могут быть недовольны неточной статистикой → дисклеймер + возможность коррекции в фазе 6
