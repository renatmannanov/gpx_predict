# Фаза 4: Веб-дашборд гонок (read-only)

## Цель

Простая веб-страница, где можно посмотреть все горные гонки Алматы: календарь, результаты прошлых лет, поиск по имени, статистика. Без авторизации, без Strava — только публичные данные.

Это НЕ полный портал (фаза 9), а быстрая "морда" для данных, которые уже есть.

## Контекст

- Все результаты спарсены (фаза 3)
- API endpoints уже есть (`backend/app/api/v1/routes/races.py`):
  - `GET /api/v1/races` — календарь
  - `GET /api/v1/races/{race_id}` — карточка гонки
  - `GET /api/v1/races/{race_id}/{year}/results` — результаты + статистика
  - `GET /api/v1/races/{race_id}/{year}/search?name=...` — поиск
- Бот уже работает для тех же данных — дашборд это альтернативный UI

## Что показывает

### Главная: Календарь гонок 2026
- Список горных гонок с датами, типом, основными параметрами
- Для прошедших — ссылка на результаты
- Для будущих — ссылка на регистрацию
- Шоссейные — дата + ссылка (без глубокой карточки)

### Карточка гонки
- Название, дата, локация
- Дистанции: км, набор, старт/финиш высота, грейд (цветовая шкала)
- Ссылка на регистрацию (если будущая)

### Результаты (по годам)
- Таблица результатов (место, имя, время, категория, клуб)
- Поиск по имени ("Найди себя")
- Статистика: финишёры, лучший, медиана, худший
- Распределение времён (бакеты с визуализацией)
- Перцентиль при поиске ("ты в топ-28%")

### Базовый прогноз (без авторизации)
- "Введи темп на плоскости → получи прогноз на дистанцию"
- Используется существующий `POST /api/v1/races/{race_id}/predict`
- Показать прогнозированное время + в каком бакете окажешься

## Чем это НЕ является

- Не полный портал (без auth, профилей, клубов, персонализации)
- Не замена бота (бот даёт персональные прогнозы через Strava)
- Не SPA с рутингом — может быть максимально простым
