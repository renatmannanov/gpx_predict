# ayda.run — Техническое задание на разработку

## Общее

**Продукт:** ayda.run — аналитический портал для бегунов Алматы. Фокус на трейл и скайраннинг, шоссейные бегуны как часть сообщества.

**Дизайн-система:**
- Background: `#252628`
- Accent / Primary: `#E8622A` (оранжевый)
- Trail accent: `#E8622A`
- Road accent: `#5B8DEF`
- Text: `#EDEEF0`
- Dim / muted: `#7A8394`
- Card bg: `rgba(255,255,255,0.04)`
- Border: `rgba(255,255,255,0.08)`
- Font: **Manrope** (400, 500, 600, 700, 800, 900)
- Border radius: 12–14px для карточек, 8px для кнопок/тегов
- Все страницы — тёмная тема, без светлого режима

**Навигация (общая для всех страниц):**
- Фиксированный nav 56px, `backdrop-filter: blur(16px)`, `background: rgba(37,38,40,0.95)`
- Лого: `ayda.run` где `.run` — акцентный цвет
- Ссылки: Гонки / Рейтинг / Клубы / Расписание
- Справа: кнопка «Войти» — оранжевая, `border-radius: 7px`

**Референсный файл:** `ayda-homepage-v2.html` — использовать как основу для компонентов и стилей.

## 1. Homepage (`/`)

**Цель страницы:** Первое знакомство с платформой. Объяснить что такое ayda.run, показать ключевые цифры сезона, направить в нужный раздел.

### Hero-секция

- Двухколоночный layout: левая колонка — текст, правая — визуальный элемент
- Заголовок крупный (~48–56px, font-weight 900, letter-spacing -2px): например «Аналитика горного бега Алматы»
- Подзаголовок (~16px, цвет dim): одна строка о том что делает платформа
- Две кнопки: «Открыть рейтинг» (primary, оранжевый) и «Как это работает» (secondary, ghost)
- Правая колонка: мини-карточка с анонсом текущего сезона — год, количество гонок, участников, топ бегун

### Секция статистики сезона

- 4 плашки в ряд: Бегунов / Гонок / Клубов / Стартов суммарно
- Каждая: большое число (акцентный цвет), подпись серым
- Фон: `rgba(255,255,255,0.04)`, разделители `1px solid var(--border)`

### Секция «Топ бегуны сезона»

- Заголовок секции + ссылка «Весь рейтинг →»
- Таблица/список: топ-5 бегунов
- Каждая строка: место (с медалью для 1–3), аватар-инициалы, имя, клуб, перцентиль, количество гонок
- Клик на строку → страница бегуна (пока заглушка)

### Секция «Клубы»

- Заголовок + «Все клубы →»
- Горизонтальный скролл или сетка 3 колонки: карточки клубов
- Каждая карточка: аббревиатура-аватар, название, специализация (⛰ трейл / 🏙 шоссе / микс), средний перцентиль, количество бегунов
- Бар специализации трейл/шоссе внизу карточки (оранжевый + синий)

### Секция «Ближайшие гонки»

- Заголовок + «Все гонки →»
- 3 карточки ближайших стартов: название, дата, дистанция, тип (trail/road), статус (открыта регистрация / скоро)
- Тип трейл — оранжевая метка, шоссе — синяя

### Footer

- Лого + короткий слоган
- Ссылки: Рейтинг / Клубы / Расписание / Гонки
- Копирайт

## 2. Season Rankings (`/season`)

**Цель страницы:** Показать рейтинг всех бегунов по итогам сезона. Дать возможность фильтровать по гонкам, найти себя, сравнить клубы.

### Шапка страницы

- Бейдж: «Алматы · Сезон 2025»
- Заголовок: «Кто лучший бегун Алматы?»
- Подзаголовок: краткое объяснение метрики перцентиля

### Switcher сезона

- Dropdown-кнопка «Сезон 2025 ▾» — при клике список годов (2022–2026)
- Текущий год помечен «сейчас»
- При смене года — данные обновляются

### Filter Bar (Linear-style)

Компактная строка фильтров, не занимает много места:

**Сегменты гонок:**
- Кнопки: «Все 30» / «⛰ Трейл» / «🏙 Асфальт»
- При клике — сразу применяется

**Кнопка «Выбрать гонки ▾»:**
- Открывает dropdown-модал
- Внутри: две колонки — Трейл (левая) / Шоссе (правая)
- Каждая гонка: чекбокс, название, дата, км
- Кнопки в шапке модала: Все / Трейл / Асфальт / Сброс
- Закрытие по клику вне или кнопкой
- Если выбраны не все — кнопка подсвечивается, показывает бейдж с количеством

**Кнопка «🔍 Найти себя»:**
- Открывает модал поиска по имени
- Fuzzy-поиск: вводишь имя → видишь совпадения
- Каждый результат: аватар-инициалы, имя, клуб, перцентиль, кол-во гонок
- Мультиселект — можно выбрать несколько вариантов написания имени
- «Применить» — выбранные строки подсвечиваются в таблице оранжевой рамкой
- Выбор сохраняется в `localStorage` ключ `ayda_mynames`
- Состояние кнопки: «Нурлан +2 ✕» когда что-то выбрано

**Текст-саммари:** «Выбрано X из 30 гонок» — появляется при кастомном выборе

### Пузырьковый график (главный визуал)

- Canvas-элемент, ширина 100% контейнера, высота ~420px
- Ось Y: перцентиль 0–100%
- Ось X: количество гонок
- Каждый пузырь = один бегун
- Размер пузыря = количество гонок
- Цвет: по зоне (см. ниже)
- 4 зоны с подписями:
  - **Элита** (перцентиль >85%) — оранжевый
  - **Продвинутые** (70–85%) — золотой `#D4A742`
  - **Середина** (50–70%) — синий `#5B8DEF`
  - **Участники** (<50%) — серый
- Hover на пузырь → тултип: имя, клуб, перцентиль, гонок
- Клик → выделение строки в таблице ниже

### Таблица рейтинга

- Колонки: Место / Имя / Клуб / Перцентиль / Гонок / Лучшая гонка / Тренд
- Место 1–3: эмодзи медали
- Аватар-инициалы перед именем, серый круг
- Перцентиль — крупно, цвет по зоне
- Тренд — ↑+3% зелёный / ↓-2% оранжевый / → серый
- «Лучшая гонка» — название гонки серым
- Строки выделенных через «Найти себя» — оранжевая рамка `outline: 1px solid #E8622A`
- Пагинация или infinite scroll (решение на усмотрение агента)

### Блок «Клубы сезона»

Две карточки рядом:

**Карточка 1: Клубы сезона**
- Список клубов, отсортированных по среднему перцентилю
- Каждая строка: ранг, аватар-инициалы, название, кол-во бегунов, прогресс-бар, перцентиль %
- Показывать топ-6

**Карточка 2: Состав участников**
- Пол: М / Ж — проценты + прогресс-бары (оранжевый / синий)
- Возраст: 18–29 / 30–39 / 40–49 / 50+ — проценты + цветные бары
- Данные реагируют на выбранные фильтры гонок

## 3. Clubs Overview (`/clubs`)

**Цель страницы:** Сравнить все клубы Алматы по характеру и результатам сезона.

### Шапка

- Бейдж: «Алматы · Сезон 2025»
- Заголовок: «Клубы Алматы»
- Подзаголовок: «Сравнение всех беговых клубов по сезонным стартам»

### Стат-стрип

4 плашки: Активных клубов / Уникальных бегунов / Гонок в сезоне / Стартов суммарно

### Пузырьковый график клубов («Карта клубов сезона»)

- Canvas, ширина 100%, высота 360px
- **Ось X:** специализация — 0% = чистое шоссе, 100% = чистый трейл
- **Ось Y:** средний перцентиль клуба
- **Размер пузыря:** количество бегунов в клубе
- **Цвет пузыря:** оранжевый если трейл >60%, синий если шоссе >60%
- **4 зоны** с лёгкой заливкой и подписями:
  - Шоссе-элита (левый верх)
  - Трейл-элита (правый верх)
  - Универсалы (середина верх)
  - Развивающиеся (низ)
- Разделительные пунктирные линии по центру осей
- Подпись под каждым пузырём: название клуба
- Hover → тултип: название, тип, бегунов, перцентиль, гонок
- Glow-эффект на пузырях через `shadowBlur`

### Рейтинг клубов (карточки)

- Заголовок + сортировка: кнопки «Перцентиль / Бегуны / Старты / Трейл%»
- Сетка карточек: `grid-template-columns: repeat(auto-fill, minmax(300px, 1fr))`
- Каждая карточка:
  - **Шапка карточки:** аватар-инициалы (40×40, border-radius 10px), название, тип клуба, ранг «#1 в сезоне» справа
  - **Бар специализации:** двухцветный (оранжевый трейл % + синий шоссе %), высота 4px
  - **Подпись к бару:** «⛰ X% трейл» и «Y% шоссе»
  - **Метрики (3 колонки):** Перцентиль с трендом (↑/↓/→) / Бегунов / Гонок
  - **Футер карточки:** «Лучший: Имя Ф.» + стрелка →
  - Hover: `translateY(-2px)`, светлее граница
- Клик → страница клуба `/clubs/:id`

### Поведение сортировки

При смене сортировки — карточки перерендериваются в новом порядке (без анимации достаточно).

## Общие технические требования

### Стек
- React + TypeScript (предпочтительно) или vanilla HTML/CSS/JS
- Никаких внешних UI-библиотек — только кастомные компоненты
- Canvas API для всех графиков (Chart.js не использовать)
- Google Fonts: `Manrope` через `<link>`

### Данные
- На первом этапе — моковые данные прямо в коде (константы)
- Структуры данных описать как TypeScript-интерфейсы
- Вся логика фильтрации — на клиенте

### Интерактивность
- Все dropdown-меню: закрываются по клику вне (`document.addEventListener('click', ...)`)
- Тултипы на Canvas: `position: absolute` внутри relative-обёртки, `pointer-events: none`
- localStorage: сохранять выбор «Найти себя» под ключом `ayda_mynames`
- Canvas: пересчитывать при `window.resize`, учитывать `devicePixelRatio`

### Адаптивность
- Breakpoint: 720px
- Mobile: скрыть nav-ссылки, уменьшить padding, одна колонка вместо двух
- Canvas-графики: адаптировать высоту и отступы

### Анимации
- Появление страницы: `opacity 0→1 + translateY(12px→0)`, длительность 0.3s
- Hover на карточках: `translateY(-2px)`, transition 0.15s
- Открытие модалов: `scale(0.96→1) + opacity`, 0.2s
