# Фаза 9: Полный веб-портал

## Цель

React-портал "Ayda — горы Алматы" — единая морда для всего: маршруты, гонки, клубы, прогнозы, профиль. Три слоя: публичный (без регистрации), персональный (Strava/Garmin), клубный (ayda_run).

## Контекст

Из agent_brief.md (секции 2, 5):

**Слой 1 (публичный, SEO):**
- Каталог маршрутов/вершин (фаза 8)
- Календарь гонок + результаты (фазы 3-4)
- Калькулятор времени без регистрации
- Регулярные события (фаза 7)

**Слой 2 (персональный):**
- "Предскажи своё время" — главный хук для подключения Strava
- "Готов ли ты к [вершине]?"
- "Твои маршруты" — какие маршруты уже пробежал
- Прогресс: "Кок-Жайляу: первый раз 3:20, лучший 2:38"
- "Твои горы": набор за сезон, количество маршрутов

**Слой 3 (клубный):**
- Дашборд клуба (из фазы 5)
- Управление клубом (из фазы 6)
- Лидерборды внутри клуба
- Календарь тренировок клуба
- Интересные дашборды и визуализации

## Предшественник: фаза 4 (race dashboard)

Фаза 4 — лёгкий read-only дашборд. Фаза 9 расширяет его до полного портала. Решение по стеку из фазы 4 определит основу для фазы 9.

## Авторизация

Из agent_brief.md (секция 5 — Authentication Architecture):

**Два способа входа:**
1. **Strava OAuth** (основной) — одно действие = авторизация + подключение данных
2. **Telegram Login Widget** (альтернативный) — для существующих ayda_run юзеров

**Связывание аккаунтов:** юзер может войти через Telegram ИЛИ Strava. При подключении второго — связывание в один профиль.

**Web auth:** JWT в httpOnly cookie.
