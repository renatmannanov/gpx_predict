# План рефакторинга Trail Run Calculations v3

**Дата:** 2026-01-31
**Статус:** В разработке

---

## Цель

Переработать архитектуру расчётов trail running:
1. 3 базовых GAP метода считают **весь маршрут**
2. Вариации (переход на шаг, персонализация, усталость) применяются **явно и раздельно**
3. Вывод **понятен и сравним** для тестирования точности формул

---

## Итоговая структура

### 3 базовых метода Trail Running

| Метод | Uphill | Downhill | Статус |
|-------|--------|----------|--------|
| `strava_gap` | Strava | Strava | Существует |
| `minetti_gap` | Minetti | Minetti | **Создать** |
| `strava_minetti_gap` | Minetti | Strava | Переименовать из текущего `minetti_gap` |

### 3 вариации

| Вариация | Описание | Фаза |
|----------|----------|------|
| Переход на шаг | После X% применяем Tobler/Naismith | Фаза 2 |
| Персонализация | Lookup из профиля + fallback | Фаза 3 |
| Усталость | +% к времени после Y часов | Фаза 4 |

### 3 варианта расчёта

| Вариант | Когда | Логика |
|---------|-------|--------|
| **1** | Нет профиля, стандартный flat | Чистая формула с flat=6:00/км |
| **2** | Пользователь ввёл свой flat | Формула × (user_flat / standard_flat) |
| **3.5** | Есть Strava профиль | Lookup + fallback на Вариант 2 |
